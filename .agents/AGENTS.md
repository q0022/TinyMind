# TinyMind Workspace Rules

## 1. การคำนวณจำนวนปุ่มลบ (Backspace Counting Rule)
* **ข้อกำหนด**: ในการสั่งลบข้อความเดิมเพื่อแทนที่ด้วยคำใหม่ (Autocorrection/Hotkey Swap) ในไฟล์ภาษา Dart/Flutter (เช่น `lib/main.dart`) **จะต้องคำนวณจำนวน backspaces โดยใช้ Grapheme Clusters เสมอ**
* **รูปแบบการเขียน**: ใช้ `.characters.length` แทนการใช้ `.length` ธรรมดา สำหรับภาษาที่มีอักษรซ้อน/สระวรรณยุกต์ลอย เช่น ภาษาไทย
* **เหตุผล**: ระบบ macOS และแอป Chromium-based (Chrome, Slack, VS Code) จะลบตัวอักษรที่มีสระซ้อน (เช่น "ที่", "ยัง") ด้วยปุ่ม Backspace เพียงครั้งเดียว หากนับเป็น Code Point ความยาวจะเกินจริงและลบล้ำไปโดนเว้นวรรค (Space) ข้างหน้า ทำให้คำติดกันเป็น `samsungtab`

## 2. การควบคุมความเร็วและคีย์บอร์ดดีเลย์ใน Swift (Swift Input Delay Constraints)
* **ข้อกำหนด**: ห้ามใส่ค่าดีเลย์ `usleep` ที่สูงเกินไปในฟังก์ชัน `replaceText` ในฝั่ง macOS Native Swift (`AppDelegate.swift`) เด็ดขาด
* **เกณฑ์จำกัด**:
  * ระยะการดีเลย์ระหว่างส่ง Backspace แต่ละปุ่ม ห้ามเกิน `usleep(1000)` (1ms)
  * ห้ามใส่การหน่วงเวลารอการสลับภาษา (`usleep(15000)`) หลังจากสั่ง `switchKeyboardLayout`
* **เหตุผล**: การใส่ดีเลย์สูงเกินไปใน Event Tap จะทำให้เธรดการประมวลผลแป้นพิมพ์ถูกบล็อก หรือเปิด-ปิดตัวดักสัญญาณ (Event Tap Enable) ไม่ทันกับเหตุการณ์จริง ส่งผลให้ผู้ใช้พิมพ์ข้อความปกติแล้วเกิดตัวอักษรสะกดซ้ำหรือปุ่มเบิ้ลหน้า (เช่น "ยัยัง", "ทีที่จะ", "แมแม้")
